API & Database

Securely Configure Gemini & Supabase

No backend editing required—enter keys through the extension UI and let the app validate and persist them locally.

Gemini AI Setup (Required)

The extension relies on Gemini 2.5 Flash for content processing, question generation, and evaluation.

  1. Visit aistudio.google.com, sign in, and create a Gemini API key.
  2. Open the extension dashboard → Settings.
  3. Paste the Gemini key into the Gemini AI – API Key field and save.
  4. The UI validates the key, stores it securely in Chrome storage, and unlocks AI analysis.

Invalid keys prompt a clear error so users can retry without guessing.

Supabase Setup (Optional for Cloud Sync)

Enable Supabase for cloud storage, device sync, and scalable analytics.

  1. Sign up at supabase.com and create a new project.
  2. From Project Settings → API, copy your Project URL and Anon Public Key.
  3. In the extension Settings, paste both values, save, and allow the system to validate.
  4. The app confirms the connection and registers a device ID for syncing future data.

Without Supabase, the extension still tracks locally, but cloud features remain locked.

Security Design

  • Chrome local storage retains all keys; nothing is hardcoded.
  • Supabase uses anon keys with Row Level Security policies.
  • User-controlled configuration eliminates external servers.
  • The app validates every key before enabling the corresponding mode.

System Modes

  • Local Mode: Gemini configured, Supabase disabled. AI analysis runs locally.
  • Cloud Mode: Both Gemini and Supabase configured. Full sync and analytics.
  • Basic Mode: Neither configured. Manual tracking only.

Start in local mode while you gather Supabase credentials, then switch to cloud mode when ready.
